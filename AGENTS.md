# 인하우스 클라이언트 프로젝트 지침

이 저장소는 pnpm과 Turbo를 사용하는 모노레포이며, shell과 remote 앱을 Module Federation으로 연결한다.

## 프로젝트 전용 규칙

- 이 저장소의 구현, 수정, 리팩터링, 코드 리뷰, 디버깅, 테스트 또는 구조 설계처럼 프로젝트 맥락이 필요한 작업을 시작하기 전에 `inhouse-skill`을 사용한다.
- `.agents/skills/*` 아래의 저장소 스킬을 생성, 수정, 리팩터링 또는 리뷰할 때 `repo-skill-linter`를 사용하고 변경 후 lint를 실행한다. 결과 품질을 비교해야 하는 의미 변경에는 추가로 `repo-skill-benchmark`를 사용한다.
- 스킬이 안내하는 참조 문서 중 현재 작업에 해당하는 문서를 모두 읽고 적용한다.
- 여러 영역이 겹치면 하나만 고르지 말고 관련 규칙을 함께 적용한다.
- 저장소와 무관한 일반 개념 질문, 단순 대화 또는 프로젝트 파일을 해석할 필요가 없는 작업에는 스킬을 불필요하게 사용하지 않는다.
- 코드 변경을 마치면 스킬의 검증 규칙에 따라 변경 범위에 맞는 타입 검사, 린트, 포맷 및 테스트를 수행한다.

## 기본 주의사항

- 패키지 관리에는 `pnpm`을 사용한다.
- 자동 생성 파일인 `routeTree.gen.ts`는 직접 수정하지 않는다.
- 공용으로 사용할 수 있는 구현은 기존 `packages/*` 워크스페이스 패키지를 먼저 확인한다.
- 사용자 변경 사항과 현재 작업 범위 밖의 파일을 임의로 되돌리지 않는다.
